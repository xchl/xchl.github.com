Redis之字符串类型(string)

By xcl

5月 6 2015 更新日期:5月 6 2015

文章目录

KEYS pattern
pattern 支持glob风格通配符。

KEYS 命令需要遍历Redis所有的键，数据较多时谨慎使用

EXISTS key //判断一个键是否存在

DEL key //删除一个或多个键 del foo bar // 2

TYPE key //返回键值的数据类型
string 字符串
hash 散列类型
list 列表类型
zset 有序集合类型

SET key value // set foo 1
GET key // get foo
INCRBY num increment // set foo 1 INCRBY foo 2 //3
INCR num //set foo 1 incr foo //2 （当操作的键不存在时默认值0，自增后为1，值为字符串会报错）
incr为原子操作无论多个客户端连接不会出现递增错误。
INCRBYFLOAT key increment //递增一个双精度浮点数
DECR key //递减
DECRBY //递减

生成自增ID(demo)
INCR count (用来自增计数)
SET user:count ‘aaa’

APPEND key value //向键值的尾部追加value ，如果键不存在这set 返回值是追加字符串的总长度
STRLEN key //返回指定键的值的长度 ut8 中文3占3个字符
MSET //设置多个键值 MSET key1 v1 key2 v2
MGET //获取多个键值 MGET key1 key2
