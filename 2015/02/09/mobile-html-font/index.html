上面的方法简单暴力。但是这样做首先是与响应式设计相违背的，对各种屏幕大小的显示也是很不友好。遂放弃 ……

提到响应式就不得百分比。话说做响应式主要靠百分比（个人的理解啊，选择性相信就好）。
css 中的长度单位可以分为两种相对长度单位（px,em,rem,百分比）和绝对长度单位（pt,pc,in,cm,mm,pc），绝对长度单位一般比较少用到。

px 表面上比较容易理解，实际上px是与屏幕分辨率由有关的。我理解的也不是很透彻就不露怯了。推荐大家看下移动前端开发之viewport的深入理解

em,rem,百分比是响应式我们现在用到的方案。em和百分比是一样的，它们相对于它们的父元素。rem表示的也是百分比只不过是相对于根元素推荐这样方式（注:IE 6,7,8不兼容）。
